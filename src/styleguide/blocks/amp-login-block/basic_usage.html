Above is an example of a "Login" block.

This block is loaded from a form-def.def.json file by the 'amp-block-loader' directive.
It should be the default way of loading it.

Example

In "form-def.def.json" file, in any array of blocks: 

            
    {
        "name": "login",
        "blockType": "AmpLoginBlockComponent",
        "blockLayout": "INLINE",
        "commonBlock": true,
        "path": "modules/amp-login-block/components/amp-login-block/amp-login-block.component",
        "custom": {
            ...
        }
    }

Default behaviour

  • 'userId', 'password' are required.

Customisation

Through the "custom" section, you have to define:

  • The title of the block: 
            
    {
        "custom": {
            "blockTitle": "To get started, enter your My AMP username and password"
        }
    }
  • IDs of your model values: 
            
    {
        "custom": {
            "controls": [
                {  "id": "userId",
                        "label": "Username",
                        "requiredMsg": "Username is a required field"
                },
                {  "id": "password",
                        "label": "Password",
                        "requiredMsg": "Password is a required field"
                },
            ]
        }
    }

In case you need more customisation options, please get in touch with DDC team!
