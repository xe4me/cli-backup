Above is an example of a "Basic Info" block.

This block is loaded from a form-def.def.json file by the 'amp-block-loader' directive.
It should be the default way of loading it.

Example

In "form-def.def.json" file, in any array of blocks: 

            
    {
        "name": "basicInfo",
        "blockType": "AmpBasicInfoBlockComponent",
        "blockLayout": "INLINE",
        "commonBlock": true,
        "path": "modules/amp-basic-info-block/components/amp-basic-info-block/amp-basic-info-block.component",
        "custom": {
            ...
        }
    }

Default behaviour

  • 'Title', 'First name', 'Last name' and 'Date of birth' are required.
  • There is a date format validator on the 'Date of birth' field (DD/MM/YYYY)

Customisation

Through the "custom" section, you have to define:

  • The title of the block: 
            
    {
        "custom": {
            "blockTitle": "Tell us about yourself"
        }
    }
  • IDs of your model values: 
            
    {
        "custom": {
            "controls": [
                { "id": "myTitle" },
                { "id": "myFirstName" },
                { "id": "myMiddleName" },
                { "id": "myLastName" },
                { "id": "myDateOfBirth" }
            ]
        }
    }

In case you need more customisation options, please get in touch with DDC team!
