Above is an example of a "SMS" block.

This block is loaded from a form-def.def.json file by the 'amp-block-loader' directive.
It should be the default way of loading it.

Example

In "form-def.def.json" file, in any array of blocks: 

            
    {
        "name": "smsMessage",
        "blockType": "Amp",
        "blockLayout": "INLINE",
        "commonBlock": true,
        "path": "blocks/amp-save-close-block/components/amp-sms-block/amp-sms-block.component",
        "custom": {
            ...
        }
    }

Default behaviour

  • 'mobileNumber', 'acceptPrivacy'.
  • There is a format validator on the 'mobileNumber' field (04nnnnnnnn) for Australian only
  • 'description' - default text : Enter your mobile number below to receive an SMS code so you can pick up from where you left off.
  • 'checkboxLable' - default text : I have read the privacy information and confirm the statements in the \'Declarations when applying for an AMP Bett3r Account\' section

Customisation

Through the "custom" section, you have to define:

  • The title of the block: 
            
    {
        "custom": {
            "blockTitle": "Just in case you want to return..."
        }
    }
  • IDs of your model values:
  • acceptPrivacy - add label to overwrite default checkout label
            
    {
        "custom": {
            "controls": [
                { "id": "mobileNumber" },
                { "id": "acceptPrivacy","label" : "I have read the  privacy information and confirm the statements, test label" },
            ]
        }
    }

In case you need more customisation options, please get in touch with DDC team!
